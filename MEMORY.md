## ⚙️ ERP 발주 자동화 (자비스 프로젝트)

### 확정된 환경 정보
| 항목 | 내용 |
|------|------|
| ERP | 경영박사 (설치형) |
| 거래처 수 | 약 50개 |
| 품목코드 수 | 약 50개 |
| ERP 입력 방식 | **품목코드만 입력하면 됨** ← 중요! |
| 발주 채널 | SMS 80%, 카카오톡 20% |
| 일 발주 건수 | 약 30건 |
| 거래처 표현 | 제각각 (예: "닭", "생닭", "9호", "후라이드용" → 모두 D001) |

### 확정된 기술 스택
| 구분 | 기술 |
|------|------|
| 서버 | Python 3.12 + FastAPI |
| DB | SQLite (초기) → MySQL (확장) |
| AI | OpenAI API |
| ERP 자동화 | pywinauto / Windows UI Automation |
| 엑셀 | pandas + openpyxl |
| SMS 수집 | Tasker 또는 MacroDroid (안드로이드) |

### 핵심 설계 원칙
- **승인 단계 필수**: AI가 자동 분석 → 담당자 확인 → 그 다음 ERP 입력
- **별칭 사전이 핵심**: 거래처별 표현과 ERP 코드를 매핑하는 테이블
- **성공 70%는 데이터 정리**: 품목 마스터 + 거래처 별칭 테이블

### 처리 흐름 (확정)
```
SMS / 카카오톡
→ AI 주문 분석
→ 품목코드 변환
→ 주문 검수 화면 (담당자 확인)
→ 경영박사 ERP 자동 입력
→ 엑셀 저장
→ 발주 완료
```

## 💰 머니핏 (MoneyFit) 프로젝트

### 브랜드 확정 사항
| 항목 | 내용 |
|------|------|
| 정식 브랜드명 | **머니핏 (MoneyFit)** |
| 별명 (애칭) | **머핏** (사람들이 자연스럽게 부를 수 있도록) |
| 슬로건 방향 | "매출이 아니라 진짜 남는 돈을 알려드립니다" |
| 타겟 | 소상공인 (치킨집, 카페, 미용실 등 1~5인 사업장) |
| 비즈니스 모델 | SaaS (구독형, 월 3~10만원) |
| 목표 수익 | 월 200만원 (6개월 목표) |

### 핵심 차별점 (확정)
- 기존: 회계 프로그램 (복잡, 회계 지식 필요)
- 머니핏: **"사장님용 경영 비서"** (돈이 어디서 새는지 바로 보여줌)
- 제1원칙: **"사장님은 입력하지 않는다"** (최대한 자동화)

### MVP 핵심 기능 (확정)
- [ ] AI 대시보드 (오늘 순이익, 경영 건강점수 100점)
- [ ] 손익 분석 (원가율 % + 금액 동시 표시)
- [ ] 오늘 마감 버튼 (하루 요약 30초 완성)
- [ ] AI 자동입력 (영수증 촬영, PDF 분석)
- [ ] 재고관리

### V2 기능 (확정)
- [ ] OKPOS 연동
- [ ] 포스뱅크 연동
- [ ] 배달의민족 자동 연동
- [ ] 쿠팡이츠 자동 연동
- [ ] 통장 거래내역 연동

### 표시 방식 확정
```
원가율  34.8% (989,000원)
인건비  18.5% (526,000원)
배달수수료  16.9% (480,000원)
```
→ **% + 금액 동시 표시** (준호 요청으로 확정)

### 경쟁 분석 메모
- 더존비즈온, 케이렙 → 너무 복잡
- 삼쩜삼 → 세금에만 집중
- 캐시노트 → 그나마 유사하지만 순이익 계산이 불편
- **핵심 경쟁상대는 "귀찮음"** (앱을 안 쓰게 되는 것)
