## ⚙️ ERP 발주 자동화 / 자비스 AI 히스토리

### 대화 [7] - 자비스 오픈소스 프로젝트

**날짜**: 2025년 하반기

**시작 질문**:
> "요즘 자비스라고 오픈소스를 이용하여 만들던데 너 알아?"

**대화 흐름 요약**:
1. 자비스(JARVIS) 개념 설명 - 아이언맨 AI 비서 오픈소스 버전
2. "집 컴퓨터 자동화 봤고 나도 만들어볼 수 있을까?" 질문
3. 초급/중급/고급 난이도 설명
4. **"중급으로 거래처 발주 확인 및 자동 처리 시스템 만들고 싶어"** 방향 확정
5. 카카오톡/SMS 발주 → ERP 자동 입력 구조 설계
6. 경영박사 ERP 사용 확인 (설치형, API 없음)

**확정된 사항**:
- 하루 30건 발주 처리 자동화
- 거래처 50개, 품목코드 50개
- SMS 80%, 카카오톡 20%
- 품목코드만 입력하면 되는 구조
- pywinauto로 경영박사 화면 자동화

**기술 스택 확정**:
```
Python + FastAPI + OpenAI API + pywinauto + SQLite + pandas
```

### 대화 [9] - 자비스 제작 가능성

**시작 질문**:
> "요즘 자비스 제작 많이 하던데 나도 가능할까?"

**대화 흐름 요약**:
1. 3단계(거래처 주문 해석 + ERP 연동)까지 가능 확인
2. 전체 구조 mermaid 다이어그램으로 설명
3. 경영박사 버전에 따른 연동 방법 논의
4. 단계별 제작 과정 세세하게 설명 요청

**핵심 결론**:
- ERP 자동화 프로젝트는 충분히 혼자 가능
- 데이터 정리(품목 마스터, 별칭 테이블)가 성공의 70%
- 완전 자동화 전에 반드시 승인 단계 넣기

### 대화 [13] - 머니핏 프로젝트 (핵심)

**시작 아이디어**:
> "자영업자들은 월 매출과 통장에 남은 돈으로 순이익을 두루뭉술하게 파악하여 돈이 어디서 새는지도 모르는 사람들이 대부분이야. 그걸 해소하고자 순이익 계산과 매장 관리를 편하게 할 수 있는 프로그램을 개발하고 싶어"

**대화 흐름 요약**:

1. **시장 분석**: 더존비즈온, 캐시노트 등 분석 → 모두 복잡하거나 불편
2. **개발 결정**: "개발하자" → 프로젝트 시작
3. **이름 후보**:
   - 경영박사 AI → 기능 설명에 좋지만 감정이 약함
   - 남는장사, 돈새나?, 돈코치 → 너무 직설적
   - 조선시대 사업가 표현 탐구 (거상, 객주, 행수)
   - 줄임말 방식 탐구 (당근=당신의근처, 한화=한국화약)
   - 이담 분석 → 앱스토어 검색 약함
   - **머니핏(MoneyFit) 분석** → 상표 4.5/5, 앱검색 5/5

4. **확정**: "머니핀이라 짓고 사람들이 머핏이라고 부를 수 있도록 하자"
   → **머니핏 (MoneyFit) / 별명 머핏** 확정

**확정된 기능들**:
- 오늘 마감 버튼 (하루 요약 30초)
- % + 금액 동시 표시 (준호 직접 요청)
- OKPOS + 포스뱅크 우선 연동 (V2)
- AI 경영 건강점수 100점
- "사장님은 입력하지 않는다" 철학

**특이사항**:
- 경쟁상대 = "귀찮음" (앱 사용 안 하게 되는 것)
- 35개 가맹점을 첫 번째 테스트 고객으로 활용
